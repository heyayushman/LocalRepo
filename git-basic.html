git add file_name : used to add a specific file to the staging area

git add -A : used to add all files to the staging area

Branch:

A branch in GitHub is a separate workspace in your repository where you can safely write code, fix bugs, or experiment with new features without affecting the main, production-ready project

By creating a clone of the main project, every single person can work separately, before actually deploying to the main project, or pushing it to the main github, Each copy is called a branch

git commit : used to commit the changes to the local repository

git commit -m "commit message" : used to commit the changes to the local repository with a message

git push : used to push the changes to the remote repository

git push origin branch_name : used to push the changes to the remote repository

git pull : used to pull the changes from the remote repository

git pull origin branch_name : used to pull the changes from the remote repository

git merge : used to merge two branches

    Merge two branches in GitHub using the command line:
    1. Switch to the branch you want to merge into (e.g., main):
       git checkout main
    2. Merge the other branch (e.g., feature-branch) into the current branch:
       git merge feature-branch


git merge conflict: 
        An event that occurs when two branches have made changes to the same line in a file, or when one branch has deleted a file that another branch has modified. Git will not be able to automatically merge the changes and will mark the file as having a conflict. The user must then manually resolve the conflict by editing the file and choosing which changes to keep. Once the conflict is resolved, the user can add the file to the staging area and commit the changes to complete the merge.
